# Project Structure

## Revit.Claude.Connector.Mcp.Server
This folder contains the Claude files to create the **mcpb extension**.
-- install mcpb packager: npm install -g @anthropic-ai/mcpb
-- CD the folder and call: mcpb pack

## Revit.Claude.Bridge
- RoslynWorker: Is simple C# script compiler. This was separated from RevitClaudeBridge to avoid exception when other Microsoft.CodeAnalysis version is already loaded.
- RevitClaudeBridge: this is the plugin and the tool provider

## RevitClaudeConnection
This folder contains the code to connect to Claude API where tools can be downloaded.

## LicenceFileGenerator
- Generates a licence file based on the machine and user.
- The file should reside in the same folder as the RevitClaudeConnector.dll
- The licence should include a url to allow the connector to download tools from a private repository. e.g. "https://drive.google.com/uc?export=download&id=1AdiTkJLMHJlVLodT5AHou9ATYK8cF1ed"
- The url should point to a file (latest version) with the following info:
{
  "version": "1.4.0", // Package version to check for updates
  "zip_url": "https://drive.google.com/uc?export=download&id=1AdiTkJLMHJlVLodT5AHou9ATYK8cF1ed", // Package zip file
  "sha256": "9d9b2c1f... (hex)" // SHA256 hash of the zip file
}

## User Installation Manual to Install Revit Claude Connector
- Install Claude Client Desktop
- Install the Revit.Claude.Connector.Mcp.Server.mcpb
- - Open Claude Client Desktop
- - Go to Settings -> Extensions -> Advanced Settings
- - Select Install Extension... and select Revit.Claude.Connector.Mcp.Server mcpb file

## User Installation Manual to Install RevitClaudeConnector plugin
- Unzip RevitClaudeConnector.zip in your local Addins folder e.g. C:\Users\ibrah\AppData\Roaming\Autodesk\Revit\Addins\2025
- Copy the licence file generated by LicenceFileGenerator.exe to the same folder as RevitClaudeConnector.dll
- Restart Revit

## Create a User Account
- Create a Json file with the following setup:
{
    "UserName": "john.doe",
    "IsValid": true,
    "PackageUrl": "https://drive.google.com/uc?export=download&id=1O7UejqCiRifenlsNi2QiGqRDeVd9npc-"
}
- Save the file as [UserAccount].json and upload and share the link in the licence file.

## Tools Package
- Create a zip file with the following structure:
revit-2025/
└── packages/
    └── Package1/
        └── versions/
            ├── 1.0.0/
            │   ├── runners/      (for the DLLs)
            │   ├── schemas/      (for the tool schemas)
            │   └── manifest.json
            ├── 2.0.0/
            └── current.txt       (points to the active version, typically the latest)
            
revit-2026/

- Check BundleSample.zip for a sample package

## Protection
obfuscar.console obfuscar.xml

## Plugin Deployment Steps
- Building the connector and run obfuscar.console obfuscar.xml
- Copy the files to ...\PluginFiles\RevitClaudeConnector
- increase version in plugin_settings.json
- zip PluginFiles and replace RevitClaudeConnector.zip
- Update version in latestPlugin.txt to match that in the plugin_settings.json

## Tools Deployment Steps
- Building the Tool project and run obfuscar.console obfuscar.xml
- Building the DMBIMRevitQAQC project and run obfuscar.console obfuscar.xml
- Copy the files to the packages folders runner and also copy Schemas and manifest.json
- zip PluginFiles and replace GlobalBundle.zip
- Update version in latest.txt

## Adding a user Steps
- Under user folder create a new account using username e.g. IFahdah.json 
- Edit the file set the UserName and set IsValid to true and the PackageUrl that contains the tools for that user
- The PackageUrl should point to a file with the version and the zip_url info
